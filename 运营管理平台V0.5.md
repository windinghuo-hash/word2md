## 文档概述

### 编写目的

本文档旨在详细描述运营管理平台的产品需求，为产品设计、开发、测试提供明确的指导依据。文档包含功能需求、业务流程、界面原型说明等内容，可直接用于开发实施。

平台涵盖API、SaaS产品全生命周期管理、灵活的计量计费体系、以及日常运营支撑功能。

### 术语解释

| 术语    | 说明                                                                                                                                                                                                                         |
| ----- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 账单金额  | 未叠加平台折扣、客户折扣等优惠的销售价格。                                                                                                                                                                                                      |
| 应付金额  | 通过现金账户余额渠道直接支付的金额。                                                                                                                                                                                                         |
| 欠费金额  | 已出账、未结清的金额。                                                                                                                                                                                                                |
| 可开票金额 | 应付金额 - 代金券抵扣金额。                                                                                                                                                                                                            |
| SKU   | SKU是商品规格下具体可售卖的最小单元，由多个商品属性（如版本、用户数、时长等）组合而成，以实现灵活的多维度定价。<br>举例：<br>商品：就像一件“外套”，是用户在商店里看到的一个独立产品（例如：某款企业软件）。<br>规格：可以理解为“尺码”，是商品的分类（例如：基础版、专业版）。<br>SKU：就是最终拿去买单的那件“具体确定的衣服”。例如，用户选择了“专业版 + 100个用户 + 一年”，这个独一无二的组合就是一个SKU。 |
| SaaS  | 这里是指在线应用软件作为商品，用户购买应用后，通过用户名+初始密码的方式使用SaaS服务。                                                                                                                                                                              |
| API   | API类是指在商家环境的在线API接口作为商品，用户可以购买API商品后，按调用次数、调用token数、字符数等进行计费。                                                                                                                                                              |

### 计费模型

| 计费类型    | 计费模式  | 说明                                                                                              | 适用产品                                          |
| ------- | ----- | ----------------------------------------------------------------------------------------------- | --------------------------------------------- |
| 包年/包月计费 | 预付费   | 商家提供周期性的服务，用户需要预先支付固定时长的费用（如一个月或一年）。服务到期前，系统会发送续费提醒；若用户继续使用，则需完成续费。                             | 适用于需求稳定的企业级应用，如OA系统、CRM系统或对tokens消耗不多的Agent等。 |
| 按需计费    | 后付费   | 商家提供先使用后付费方式，系统会根据用户的实际使用量（如API调用次数、存储空间、tokens数、视频/语音时长等）每分钟/小时/每天出账单，再从账户余额里扣款。               | 适用于用量波动大的场景。<br>API、SaaS                      |
| 资源包计费   | 预付费   | 提前购买固定额度的资源（如 1000万 Token 包，或 5000次 API 调用包），有效期内扣减，超出后转按需计费或阻断。<br>资源包可区分是否周期性重置（周期性的重置套餐包次数）。 | API、Agent增值消耗包（如Tokens）                       |
| 产品组合计费  | todo； |                                                                                                 |                                               |
